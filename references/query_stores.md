# 门店查询场景

1. **获取城市和地址** — 必须同时知道**城市**和**具体地址/地标**才能查询。
    - 如果用户只说了地址没说城市（如"环球贸易中心附近"），先问："请问您在哪个城市？"
    - 如果用户已提供城市和地址（如"广州新港东路附近吃面"），直接进行下一步。
    - 将地址组合为完整格式：`{城市}{区} + {具体地址}`，例如"广州市海珠区新港东路环球贸易中心"

2. **调用MCP工具** — 使用 `queryStoresByAddress(address)` 传入完整地址，内部自动解析经纬度并返回附近门店。
**调用示例：**
```bash
# 查询门店
node scripts/xiaonoodles_mcp.js queryStoresByAddress '{\"address\":\"广州市海珠区新港东路\"}'
```

3. 门店信息展示规范-查询到门店后，使用以下结构展示信息。每次最多展示 **5家最近营业状态为开业的门店**，按距离由近到远、开业到未开业的顺序排列。

### 门店信息卡片模板
```
📍 [门店名称]（距您约 X.X 公里）
🏠 地址：[详细地址]
⏰ 营业时间：[时段，如 10:00–22:00]，[营业状态]
📞 联系电话：[号码]
🍽️ 服务方式：[堂食 ✅ | 自提 ✅ | 外卖 ✅（自营/美团/淘宝闪购/京东）]
─────────────────────
```

### 无门店覆盖处理

当用户位置周边3公里内无门店时：

```

距您最近的门店在：

💡 您也可以：
① 在遇见小面小程序或美团/淘宝闪购/京东搜索"遇见小面"查看外卖配送范围

③ 感谢您的关注，新店开业第一时间通知您 🔔
```

## 门店字段说明

| 字段 | 说明              |
|-----|-----------------|
| id | 门店唯一编码          |
| name | 门店全称            |
| city | 所在城市            |
| district | 区域/商圈           |
| address | 详细地址            |
| phone | 联系电话            |
| hours | 营业时间            |
| dine_in | 是否可堂食(true/false) |
| delivery | 是否支持外卖(true/false) |
| delivery_platforms | 外卖支持平台 |
| status | 营业状态 |
